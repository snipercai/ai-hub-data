# 上下文记忆库

## 📋 提示词内容

```
我在做 [项目名称]。这是你需要保留的全部内容:[粘贴项目上下文]。从现在开始，在每个回应中引用这个上下文，不用我重复。总结你会记住什么。
```

## 🎯 适用场景

- 长期项目需要保持上下文一致性
- 复杂项目涉及多个相关概念和决策
- 避免在对话中重复说明背景信息
- 确保 AI 助手理解项目的完整背景
- 多轮对话中保持连贯性

## 💡 使用说明

### 步骤 1: 提供项目名称

明确说明项目的名称和性质，帮助 AI 建立初步认知框架。

### 步骤 2: 粘贴项目上下文

提供完整的项目背景信息，包括：
- 项目目标和范围
- 技术栈和架构
- 关键决策和约束
- 当前状态和进展
- 重要文档和参考

### 步骤 3: 建立记忆库

AI 将整理和总结关键信息，形成可引用的记忆库。

### 步骤 4: 持续引用

在后续对话中，AI 会自动引用上下文，无需重复说明。

## 📝 使用示例

### 示例 1

**输入**:
```
我在做电商平台的后端重构项目。这是你需要保留的全部内容：

项目目标：将单体架构拆分为微服务
技术栈：Node.js + Express -> Go + gRPC
当前状态：已完成用户服务，正在开发订单服务
关键决策：
- 使用事件驱动架构
- 数据库按服务拆分
- API 网关统一入口
约束条件：
- 必须保持向后兼容
- 不能影响现有用户
- 3 个月内完成

从现在开始，在每个回应中引用这个上下文，不用我重复。总结你会记住什么。
```

**输出**:
```
## 上下文记忆库已建立

我会记住以下关键信息：

### 项目信息
- **项目**: 电商平台后端重构
- **目标**: 单体架构 → 微服务
- **技术栈**: Node.js/Express 迁移到 Go/gRPC

### 当前进展
- ✅ 用户服务已完成
- 🔄 订单服务开发中

### 架构决策
- 事件驱动架构
- 数据库按服务拆分
- API 网关统一入口

### 约束条件
- 保持向后兼容
- 不影响现有用户
- 3 个月完成时限

在后续对话中，我会基于这个上下文提供建议，无需你重复说明。
```

### 示例 2

**输入**:
```
我在做 AI 写作助手应用。这是你需要保留的全部内容：

产品名称：WriteAI
目标用户：内容创作者、博主、营销人员
核心功能：
- AI 辅助写作
- 语法和风格检查
- SEO 优化建议
- 多平台发布
技术实现：
- 前端：React + TypeScript
- 后端：Python FastAPI
- AI 模型：集成多个 LLM
商业模式：SaaS 订阅制

请总结你会记住的内容。
```

**输出**:
```
## WriteAI 项目上下文已保存

我记住的关键信息：

### 产品定位
- **名称**: WriteAI
- **用户**: 内容创作者、博主、营销人员
- **模式**: SaaS 订阅

### 功能特性
- AI 辅助写作、语法检查、SEO 优化、多平台发布

### 技术架构
- 前端：React + TypeScript
- 后端：Python FastAPI
- AI：多 LLM 集成

后续讨论功能开发、技术选型或业务问题时，我会基于这个上下文提供针对性建议。
```

### 示例 3

**输入**:
```
我在做物联网设备监控系统。这是项目上下文：

监控对象：工厂设备（温度、压力、振动传感器）
数据量：1000+ 设备，每秒 10 万条数据
技术栈：
- 数据采集：MQTT
- 流处理：Apache Kafka + Flink
- 存储：InfluxDB + PostgreSQL
- 前端：Vue.js + ECharts
关键需求：
- 实时告警（延迟<1 秒）
- 历史数据查询
- 预测性维护

请确认你理解并会记住这些内容。
```

**输出**:
```
## 物联网监控系统上下文已建立

我理解并会记住：

### 系统规模
- 1000+ 设备，10 万条数据/秒

### 技术栈
- 采集：MQTT
- 处理：Kafka + Flink
- 存储：InfluxDB（时序）+ PostgreSQL（关系）
- 展示：Vue.js + ECharts

### 核心需求
- 实时告警（<1 秒延迟）
- 历史查询
- 预测性维护

在后续讨论中，我会基于这个工业级 IoT 场景提供建议，考虑高并发、实时性和可靠性要求。
```

## 🔧 参数调整建议

- **上下文详细程度**: 可根据项目复杂度调整，简单项目 100-200 字，复杂项目 500-1000 字
- **更新频率**: 项目重大变更时主动更新上下文
- **引用方式**: 可指定 AI 是显式引用（每次说明）还是隐式引用（直接应用）
- **记忆范围**: 可指定只记住技术细节、只记住业务逻辑或全部记住

## ⚠️ 注意事项

- 上下文信息应准确、完整、结构化
- 避免包含敏感信息（密码、密钥、个人隐私）
- 项目发生重大变更时，及时更新上下文
- 长对话中可定期回顾和确认上下文
- 如 AI 误解了某些内容，及时纠正
