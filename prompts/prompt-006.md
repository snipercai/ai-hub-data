# 错误发现者

## 📋 提示词内容

```
审查我在这个对话中所做的一切。找出：我犯的逻辑错误、可能错误的假设、我过度自信的地方、我没有考虑的风险。在我承诺这个方案前质疑我的思维。
```

## 🎯 适用场景

- 做出重要决策前的最后审查
- 完成方案设计后的质量检查
- 发现思维盲点和认知偏见
- 降低决策风险和错误概率
- 提升思维质量和判断力

## 💡 使用说明

### 步骤 1: 回顾对话

AI 会回顾整个对话历史，整理你提出的观点、方案和决策。

### 步骤 2: 识别逻辑错误

检查是否存在：
- 因果谬误
- 以偏概全
- 错误二分法
- 循环论证
- 诉诸情感等逻辑问题

### 步骤 3: 挑战假设

质疑：
- 未经验证的前提
- 基于经验的惯性假设
- 文化或环境带来的偏见
- 过度简化的模型

### 步骤 4: 指出过度自信

识别：
- 缺乏证据的确定性陈述
- 低估难度的判断
- 高估能力的评估
- 忽视不确定性的预测

### 步骤 5: 揭示未考虑的风险

发现：
- 潜在的负面后果
- 外部依赖的风险
- 极端情况的影响
- 连锁反应

## 📝 使用示例

### 示例 1

**输入**:
```
（在讨论了一个营销方案后）
审查我在这个对话中所做的一切。找出：我犯的逻辑错误、可能错误的假设、我过度自信的地方、我没有考虑的风险。在我承诺这个方案前质疑我的思维。
```

**输出**:
```
## 思维审查报告

### 🔴 逻辑错误

1. **因果谬误**: "上次活动成功是因为 KOL 推广，所以这次也要找 KOL"
   - 忽略了市场环境、产品阶段等变化
   - 可能存在其他成功因素

2. **确认偏误**: 只关注支持你观点的数据
   - 引用了 3 个成功案例，忽略 5 个失败案例

### 🟡 错误假设

1. **"预算增加 2 倍，效果也会增加 2 倍"**
   - 营销效果通常是非线性的
   - 可能存在边际效益递减

2. **"目标用户行为不变"**
   - 用户偏好可能已变化
   - 竞争对手动作可能影响

3. **"渠道质量稳定"**
   - 流量成本在上升
   - 平台算法可能调整

### 🟠 过度自信

1. **"转化率能达到 5%"**
   - 行业平均 2-3%
   - 缺乏 A/B 测试数据支持

2. **"2 周完成所有准备"**
   - 未考虑意外情况
   - 历史数据显示类似项目平均 4 周

3. **"不会有技术障碍"**
   - 新集成功能未充分评估
   - 团队相关经验有限

### ⚠️ 未考虑的风险

1. **外部风险**:
   - 平台政策变化（如广告投放规则）
   - 竞争对手反击（价格战、挖角）
   - 供应链中断

2. **执行风险**:
   - 关键人员离职
   - 技术实现延期
   - 质量问题

3. **市场风险**:
   - 用户需求变化
   - 负面舆情
   - 宏观经济影响

### 💡 建议改进

1. 进行小规模试点验证假设
2. 制定风险应对预案
3. 设置更保守的目标和时间表
4. 建立关键指标监控机制
5. 寻求第三方意见验证
```

### 示例 2

**输入**:
```
（在设计了一个技术方案后）
审查我的技术方案设计，找出问题。
```

**输出**:
```
## 技术方案审查

### 逻辑问题

1. **单一故障点**: 方案依赖单个数据库实例
   - 你说"数据库很稳定不会挂" - 这是过度自信
   - 任何硬件都可能故障

2. **容量规划错误**: "当前 10 万用户，设计 100 万容量足够"
   - 未考虑用户增长曲线
   - 未考虑峰值流量（可能是平均的 10 倍）

### 错误假设

1. **"用户请求均匀分布"**
   - 实际存在明显的峰谷效应

2. **"网络延迟可忽略"**
   - 跨地域访问延迟可能达 200ms+

3. **"团队能快速掌握新技术"**
   - 学习曲线被低估

### 过度自信

1. **"系统可用性 99.99%"**
   - 需要冗余设计和自动化运维
   - 当前方案只能达到 99.9%

2. **"开发周期 1 个月"**
   - 未考虑测试、调试、文档时间
   - 建议 2-3 个月

### 遗漏的风险

1. 数据备份和恢复策略缺失
2. 安全审计未纳入
3. 监控告警不完善
4. 回滚方案未设计
5. 合规性考虑不足
```

### 示例 3

**输入**:
```
（在规划职业发展后）
审查我的职业规划思路。
```

**输出**:
```
## 职业规划审查

### 逻辑错误

1. **线性思维**: "现在做工程师→3 年后经理→5 年后总监"
   - 职业路径不是线性的
   - 忽略了机会和选择权

2. **幸存者偏差**: "某某这样成功了"
   - 个体经验不可简单复制

### 错误假设

1. **"行业会持续增长"**
   - 行业周期波动是常态

2. **"技能不会过时"**
   - 技术更新速度在加快

3. **"大公司=稳定"**
   - 大公司也在裁员优化

### 过度自信

1. **"3 年内薪资翻倍"**
   - 依赖个人表现和公司状况
   - 市场环境影响

2. **"不会遇到职业瓶颈"**
   - 每个人都有瓶颈期
   - 需要提前准备

### 未考虑的风险

1. 行业下行风险
2. 健康和工作平衡
3. 技能单一化风险
4. 地域限制
5. 家庭因素影响

### 建议

1. 建立可迁移技能
2. 保持学习和适应性
3. 发展副业和多元收入
4. 建立人脉网络
5. 定期审视和调整规划
```

## 🔧 参数调整建议

- **审查严格度**: 可根据决策重要性调整（温和建议/严格审查/极度挑剔）
- **关注重点**: 可指定关注特定类型的问题（逻辑/假设/风险等）
- **时间范围**: 可指定审查整个对话或最近的决定
- **反馈方式**: 可选择直接指出问题或引导式提问

## ⚠️ 注意事项

- 这个提示词的目的是帮助你，不是打击你
- 被指出的问题不一定都是对的，需要理性判断
- 避免防御性心理，保持开放心态
- 重要决策建议多次审查
- 结合其他信息来源验证
