# 并行工作管理器

## 📋 提示词内容

```
我需要同时完成这些任务:[列出任务]。生成代理团队：为每个任务分配一个代理，协调它们之间的配合，报告所有代理的进度完成后合并结果。并行运行。
```

## 🎯 适用场景

- 需要同时处理多个独立或相关任务
- 复杂项目需要分工协作
- 需要跟踪多个工作流的进度
- 希望提高多任务处理效率

## 💡 使用说明

### 步骤 1: 列出任务

清晰列出所有需要完成的任务，确保每个任务都有明确的目标和交付物。

### 步骤 2: 分配代理

为每个任务指定专门的代理，说明每个代理的职责和专长领域。

### 步骤 3: 协调执行

建立代理间的沟通机制，确保任务间的依赖关系得到处理，避免冲突和重复工作。

### 步骤 4: 合并结果

收集所有代理的输出，整合成一致的最终结果，确保各部分协调一致。

## 📝 使用示例

### 示例 1

**输入**:
```
我需要同时完成这些任务：
1. 编写用户注册模块的后端 API
2. 设计用户注册页面的前端界面
3. 创建数据库用户表结构
4. 编写单元测试用例

生成代理团队：为每个任务分配一个代理，协调它们之间的配合，报告所有代理的进度完成后合并结果。并行运行。
```

**输出**:
```
## 代理团队组建完成

### 分配方案：
- **代理 A（后端专家）**: 用户注册 API 开发
- **代理 B（前端专家）**: 注册页面 UI 设计
- **代理 C（数据库专家）**: 用户表结构设计
- **代理 D（测试专家）**: 单元测试编写

### 进度报告：
- 代理 A: 已完成 API 端点设计，进行中...
- 代理 B: 界面原型已完成，等待确认...
- 代理 C: 数据库 schema 已创建...
- 代理 D: 测试框架已搭建...

### 合并结果：
所有模块已完成并集成，系统可正常运行。
```

### 示例 2

**输入**:
```
我需要同时完成这些任务：
1. 市场调研分析
2. 竞品功能对比
3. 用户需求整理
4. 产品路线图规划

生成代理团队并协调执行。
```

**输出**:
```
## 产品规划代理团队

### 团队结构：
- 市场分析代理、竞品分析代理、用户研究代理、产品策略代理

### 执行进度：
各代理并行工作中，已建立共享知识库...

### 最终交付：
完整的产品规划文档，包含市场分析、竞品对比、用户画像和路线图。
```

### 示例 3

**输入**:
```
我需要同时完成这些任务：
1. 代码重构
2. 文档更新
3. 性能优化
4. 安全审查

为技术债务清理项目生成代理团队。
```

**输出**:
```
## 技术债务清理团队

### 代理分配：
- 重构代理、文档代理、性能代理、安全代理

### 协调机制：
建立变更日志，确保各代理了解相互的修改...

### 整合结果：
代码质量提升，文档完善，性能提升 30%，安全隐患已修复。
```

## 🔧 参数调整建议

- **任务数量**: 建议 3-7 个任务，过多会导致协调复杂，过少则不需要并行处理
- **代理专长**: 根据任务性质调整代理的专业领域定义
- **报告频率**: 可根据项目复杂度调整进度报告的频率（实时/阶段性）
- **协作深度**: 简单任务可独立执行，复杂任务需加强代理间协作

## ⚠️ 注意事项

- 确保任务之间有清晰的边界，避免职责重叠
- 建立有效的沟通机制，处理任务间的依赖关系
- 定期检查进度，及时发现和解决阻塞问题
- 最终整合时要确保各部分的一致性和兼容性
- 对于高度依赖的任务，可能需要调整执行顺序
